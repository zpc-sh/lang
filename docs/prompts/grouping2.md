| Grouping                          | Roles Included                                                           | Rationale & When to Use                                                                                                                                                                                        | Example Workflow in LANG Project                                                                                                                   |
| --------------------------------- | ------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Design & Planning Swarm**       | Architect/Designer, Researcher/Innovator, Coordinator/Project Manager    | Focuses on ideation and high-level planning; Coordinator ensures alignment. Use for early stages like extending MCP or new LSP methods—prevents implementation without solid foundations.                      | Researcher proposes ML integrations (e.g., lang_ml_embed); Architect designs Ash resource structure; Coordinator assigns to Builder.               |
| **Core Development Swarm**        | Builder/Implementer, Optimizer/Performance Engineer, Documenter          | Handles building and refining code; Documenter captures inline docs. Ideal for feature implementation like MCP bridge—ensures efficient, well-documented code adhering to AshOban for async tasks.             | Builder generates diffs for proxy/router.ex; Optimizer profiles Rust NIFs for raw parsing; Documenter updates docs/mcp.md.                         |
| **Quality Assurance Swarm**       | Debugger/Tester, Security Auditor, Integrator/Deployer                   | Emphasizes testing, security, and integration; prevents regressions in networking/client flux. Use for post-implementation phases like MCP proxy testing—covers preventatives and deployment.                  | Debugger writes multi-client tests for lsp_channel.ex; Security audits AshAuthentication in MCP; Integrator updates CI for NIF deploys.            |
| **Oversight & Maintenance Swarm** | Coordinator/Project Manager, Security Auditor, Documenter                | Monitors ongoing work, ensures guardrails (e.g., from AGENTS.md), and maintains docs. Useful for long-running swarms or audits—handles client flux and ethical checks.                                         | Coordinator orchestrates agent handoffs via lang_agent_consensus; Security scans for proxy vulnerabilities; Documenter ensures protocol adherence. |
| **Innovation & Research Swarm**   | Researcher/Innovator, Architect/Designer, Optimizer/Performance Engineer | Drives new ideas with feasibility checks; Optimizer stubs performance models. Apply for exploratory tasks like advanced MCP patterns—balances creativity with practicality.                                    | Researcher explores multi-modal proxies; Architect designs integration; Optimizer benchmarks token efficiency in lang*tokens*\*.                   |
| **Full Project Swarm**            | All Roles (coordinated by Coordinator)                                   | Comprehensive group for major milestones (e.g., MCP rollout); uses LSP methods for inter-agent comms. Reserve for complex innovations—ensures holistic coverage but avoid for small tasks to prevent overhead. | Coordinator assigns subtasks: Builder/Debugger for core MCP, Security/Integrator for auth/deployment, Researcher for future extensions.            |

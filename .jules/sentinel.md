## 2024-05-11 - [Remote Code Execution via Dynamic Agent Module Dispatch]
**Vulnerability:** In `LSPComparisonWorker.execute_agent_task`, the system dynamically dispatched to agent modules based on user-controlled input (`agent_variant["provider_module"]`) using `apply(String.to_atom(agent_module), ...)`. This allowed an attacker to execute arbitrary code (RCE) by providing any module and potentially cause denial-of-service via atom exhaustion from creating infinite atoms.
**Learning:** Dynamic module dispatch from untrusted strings, even when expected to conform to specific names (like testing variants), is a major risk. Standard library function `String.to_atom/1` should never be used on external input, and `apply/3` should never use untrusted module names.
**Prevention:** Always validate that dynamic module names start with the expected, safe namespace (e.g., `Elixir.Lang.Testing.Variants.`). Furthermore, strictly use `String.to_existing_atom/1` within an isolated `try/rescue` block to ensure that only pre-compiled valid modules are loaded and to prevent masking legitimate `ArgumentError` exceptions thrown elsewhere in the app.

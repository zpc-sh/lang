## 2024-04-20 - Atom Exhaustion DoS in Oban Workers
**Vulnerability:** Untrusted user input (job arguments like `env` and `task`) from Oban jobs was being converted to atoms dynamically using `String.to_atom/1`. This is a known vulnerability in the Elixir/Erlang VM since the atom table has a hard limit (by default ~1,048,576 atoms), and atoms are not garbage collected. An attacker who can control job arguments could exhaust the atom table, crashing the VM and causing a complete Denial of Service.
**Learning:** Background jobs often process user-provided strings. Passing untrusted strings to functions that convert them to atoms without bounds checking leads to DoS vulnerabilities. `String.to_atom/1` should never be used with unbounded, unsanitized input.
**Prevention:** Always use `String.to_existing_atom/1` for parsing external string data to atoms, such as worker task arguments. Wrap the call in a `try/rescue ArgumentError` block to handle cases where the input is invalid or hasn't been instantiated, logging a warning and returning securely without failing entirely or exposing internals.
